- Shaders are like a function call

A Simple Vertex Shader

```C
#version 330 core

in vec4 vPosition;
in vec4 vColor;

out vev4 color;

uniform mat4 ModelViewProjectionMatrix;

void
main()
{
	color = vColor;
	gl_Position = ModelViewProjectionMatrix * vPosition;
}
```

- Data is copied into the shader from OpenGl through the `in` variables
- Values in those variables are updated every time OpenGL executes the shader
- `uniform` variable do not change per vertex or fragment, has the same value across geometric primitives, until the application updates them

# Computational Invariance

- Does not guarantee that two identical computations in different shaders will result in exactly the same value
- GLSL has 2 methods for enforcing this type of invariance between shaders using
	- `invariant`
	- `precise`
- Both methods will cause computations done by the graphics device to create reproducibility
- Does not help reproduce the same results between the host and the graphics device

**The `invariant` Qualifier**

- Applied to any shader output variable
- Guarantee that if two shader invocations each set the output variable with the same expression and the same values

```c
invariant gl_Position;
invariant centroid out vec3 Color:
```
- Impose invariance on all varying variables in shader

`#pragma STDGL invariant(all)`

- Guaranteeing invariance usually disables optimizations that may have been performed by the GLSL compiler

**The `precise` Qualifier**

- Applied to any computer variable or function return value
- Not to increase precision, but to increase reproducibility of a computation
- Use in tessellation shaders to avoid forming cracks in geometry
- Use to get the same result from an expression, even if values feeding the expression are permutes in a way that should not mathematically affect the result

```c
precise gl_Position;
precise out vec3 Location;
precise vec3 subdivide(vec3 P1, vec3 P2){...}
```
- `fma()` allow to mix instruction types

```c
precise out float result;
...
float f = c * d;
float result = fma(a, b, f);
```

# Interface Blocks

- Shader variables are shared with the application or between stages can be organized into blocks of variables
- Uniform variables can be organized into `uniform` blocks, input and output variables into `in` and `out` blocks, and shader storage buffers into `buffer` blocks


```c
uniform b {
	vec4 v1;
	bool v2;
};

uniform b {
	vec4 v1;
	bool v2;
} name;
```

- The block interface name `b` is used for interface matching or external identification
- The name at the end `name` is used in the rest of the shader for accessing members

# Buffer Blocks

- Shader storage buffer object operate similar to uniform blocks
- Shader can write to them, modifying their content as seen from other shader invocations or the application
- Size can be established just before rendering, rather than at compile or link time

```c
buffer BufferObject {
	int mode;
	vec4 points[];
};
```

- Writes modifying the shader storage buffer object will be visible to other shader invocations
- If you don't need to write to a buffer, use a uniform block, as your device might not have as many resources available


# In/Out Blocks

# GLSL Subroutine Setup

3 Steps to set up the pool of subroutines
1) Define the subroutine type using the subroutine key word

`subroutine returnType subroutineType(type param, ...);`

2) Using the _subroutineType_, define the set of subroutines to dynamically select from using the subroutine keyword

`subroutine (subroutineType) returnType functionName(...);`

3) Specify the subroutine uniform variable that will hold the "function pointer"

`subroutine uniform subroutineType variableName;`

# Selecting Shader Subroutines

- A subroutine uniform value was declared, and we will need its location in order to set its value

`GLint glGetSubroutineUniformLocation(GLuint program, GLenum shadertype, const char* name)`
- Returns the location of the subroutine uniform name in program for shading stage specified by shader-type

`GLuint glGetSubroutineIndex(GLuint program, GLenum shadertype, const char* name)`
- Returns the index of the shader function associated with name from program for the shading stage specified by the shader-type

`GLuint glUniformSubroutinesuiv(GLenum shadertype, GLsizei count, const GLuint * indices)`
- Sets count shader subroutine uniform using the values in indices, for the shader stage shader-type

```c
GLint materialShaderLoc;
GLuint ambienIndex;
GLuint diffuseIndex;

glUseProgram(program);

materialShaderLoc = glGetSubroutineUniformLocation(
	program, GL_VERTEX_SHADER, "materialShader");
)

if (materialShaderLoc < 0){
	// Error
}

ambientIndex = glGetSubrutineIndex(program, GL_VERTEX_SHADER, "ambient");

diffuseIndex = glGetSubroutineIndex(program, GL_VERTEX_SHADER, "diffuse");

if (ambientIndex == GL_INVALID_INDEX || diffuseIndex == GL_INVALID_INDEX){
	// Error
}
else {
	GLsize i n;
	glGetIntegerv(GL_MAX_SUBROUTINE_UNIFORM_LOCATIONS, &n);

```
