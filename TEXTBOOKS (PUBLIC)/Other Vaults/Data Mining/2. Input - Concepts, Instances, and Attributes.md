# What's a Concept?

- 4 types of learning styles
	- Classification learning
	- Association learning
	- Clustering
	- Numeric prediction
- The thing to be learned is the concept and the output produced by the learning scheme is the concept description
- Multi-labeled instances are classification scenarios that result in multiple classes
- Classification learning is sometimes called supervised, and operates under supervision by being provided with the actual outcome for each of the training examples
- The success rate on test data gives an objective measure of how well the concept has been learned
- Success is measure more subjectively in terms of how acceptable the learned description is to a human user
- Association rules differ from classification rules in two ways
	- They can predict any attribute
	- They predict more than one attribute's value
- Association rules normally involve only non-numeric attributes
- The success of clustering is often measure subjectively in terms of how useful the results appears to be a human user
- Numeric prediction is a variant of classification learning in which the outcome is a numeric value rather than a category

# What's in an Example?

- Each instance is an individual, independent example of the concept to be learned
- Instances are characterized by the values of a set of predetermined attributes
- Each dataset is represented as a matrix of instances vs. attributes, which in  database terms is a single relation, or a flat file

## Relations

- The idea of specifying only positive examples and adopting a standing assumption that the rest are negative is called the closed-world assumption
- Most machine learning schemes have trouble with relational instances
- In database, you take two relations and join then together to make one, a process of flattening called denormalization
- Denormalization produces apparent regularities in the data that are completely spurious and are in fact reflections of the original database structure
- Abstract computational problems involve relations that are not finite
	- Ancestor-of
- List processing and logic programming that deals with infinite problems is called inductive logic programming

## Other Example Types

- Structured examples such as graphs and trees can be viewed as special cases of relations that are mapped into independent instances by extracting local or global features based on their structure and representing them as attributes
- Multi-instance problems often arise naturally when relations from a database are joined
- Classification is performed on a per-use basis, so the set of session instances associated with the same user should be viewed as a single example for learning
- The input to a data mining scheme is generally expressed as a table of independent instances of the concept to be learned


# What's in an Attribute?

- Each instance that provides the input to machine learning is characterized by its values on a fixed, predefined set of features or attributes
- Problems arise when instances have different features or the existence of one feature depends on the value of another
- The value of an attribute is a measurement of the quantity to which the attribute refers
- Numeric attributes, sometimes called continuous, are measured with real or integer values
- Nominal attributes takes on values in a pre-specified, finite set of possibilities and are sometimes categorical
	- Nominal
	- Ordinal
	- Interval
	- Ratio
- Enumerated is the standard term used in computer science to denote a categorical data type
- Discrete also has connotation of ordering because you often discretize a continuous numeric quantity
- Dichotomy, only has two members, often true and false, or yes or no; also called Boolean values
- Circular ordering could affect the kinds of test that are considered
- Metadata, data about data

## Attribute Types

- ARFF format accommodates the 2 basic data types
	- Nominal
	- Numeric
- Relation-valued attributes contain separate sets of instances that have basic attributes
- Many schemes treat numeric attributes as ordinal scales and use less-than and greater-than comparisons
- Some treat them as ratio scales and use distance calculations
- Attribute are often normalized to lie in a fixed range, between 0 and 1
- Another normalization technique is to calculate the statistical mean and the standard deviation
- Standardizing a statistical variable and results in a set of values whose mean is 0 and the standard deviation is 1
- Instance-based and regression methods deal only with ratio scales because they calculate the "distance" between two instances based on the values of their attributes
- Postal zip codes indicate areas that could be represented by geographical coordinates
- Nominal values can be coded as integers

## Missing Values

- Missing values are frequently indicated by out-of-range entries
	- -1
	- 0
	- Blank
	- Dashes
- Most machine learning schemes make the implicit assumption that there is no particular significance in the fact that a certain instance has an attribute value missing
