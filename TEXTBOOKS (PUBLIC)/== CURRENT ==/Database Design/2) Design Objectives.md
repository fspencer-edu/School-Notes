
## The Importance of Theory

- Set theory and first-order predicate logic is what allows the relational database to guarantee accurate information

## Advantage of Learning a Good Design Method

- Skills to design a source database structure
- Provides an organized set of techniques for a design process
- Keep missteps and design reiterations to a minimum
- Makes the design process easier
- Understand RDBMS application more fully and effectively

## Objectives of Good Design

- Database supports both required and ad hoc information retrieval
- Tables are constructed property and efficiently
- Data integrity is imposed at the field, table, and relationship levels
- Database supports business rules relevant to the organization
- Database lends itself to future growth

## Benefits of Good Design

- Database structure is easy to modify and maintain
- Data is easy to modify
- Information is easy to retrieve
- End user applications are easy to develop and build

## Database-Design Methods

### Traditional Design Methods

- Requirement analysis
	- Examination of business being modelled
- Data modeling
	- Data-modelling method
		- ER diagramming
		- Semantic object modelling
		- Object role modelling
		- UML modelling

	- Each table is assigned a primary key
	- Relationships are established for foreign keys


- Normalization
	 - Process of decomposing large tables into smaller one in order to eliminate redundant data and duplicate data
	 - Avoid problems with inserting, updating, or deleting
	 - Table structures are tested against normal forms and modified
		 - First Normal Form
		 - Second Normal Form
		 - Third Normal Form
		 - Normal Form
		 - Fifth Normal Form
		 - Sixth Normal Form
		 - Boyce-Codd Normal Form
		 - Domain/Key Normal Form

### The Design Method Presented in This Book

- Requirement analysis
- Simple ER-diagramming
- Does not incorporate normalization
