What is a Database?
- Collection of data that can represent an entity as a table or view of having a fixed or flexible set of attributes
- Stored electronically for easy access and management
- Better performance compared to files

What is a Database Management System (DBMS)?
- Software integrate database features
	- Create
	- Insert
	- Update
	- Delete
- Also provides support for administrative tasks
	- Managing performance
	- In-memory cache
	- Storing indexes
	- Export/Import
- Store proprietary format

DBMS Examples
- MySQL
- PostgreSQL
- Oracle
- MSSQL server
- Amazon Aurora


Types of Databases
1) Hierarchical
2) Relational
3) NoSQL


Hierarchical
- Popular around 1970
- Store data in a tree kind of structure that maps parent records to child records
- Maintain one-to-many relationships by mapping multiple child to one parent with a constraint that a child can have only one parent
- Reduces from complexity and inability to scale with data size and number of entities

Relational Databases
- 1980s
- Tabular storage structure
	- Table or view
- Simple for end users to understand the data and represent them as records and columns
- Fixed schema

	Structured Query Language (SQL)
	- Programming language that interacts with databases

- Relational databases provide PL/SQL (Procedural Language for SQL) that combines more than one SQL statement to achieve a common task or return a set of values


SQL Commands:
- Data Definition Language (DDL)
	- CREATE
- Data Manipulation (DML)
	- INSERT
	  UPDATE
	  DELETE
- Data Query Language (DQL)
	- SELECT
- Data Control Language (DCL)
	- GRANT
- Transaction Control Language (TCL)
	- ROLLBACK


NoSQL Databases
- Variable schema
- Specific query pattern for a specific type of NoSQL, database may give a better performance
	- Key Value Store
	- Document Store
	- Graph database
	- In-memory database
	- Search database

Difference Between OLTP and OLAP

- Online Transaction Processing (OLTP) and Online Analytical Processing (OLAP) are two different data processing systems
- OLTP
	- Record level processing
	- Serve front end application
	- Request response in a few seconds
	- Analyzing larger volumes of records to find insights
- OLAP
	- Column level operation
	- Serve analytical Business Intelligence (BI)
	- Response time is generally lower than OLTP
	- Record level operation that require high concurrency


- Data Lake and Data Warehousing systems are primarily leveraged for OLAP use cases]

Overview of Big Data and Distributed Processing
- Traditional database technology are not able to scale to process

What is Big Data?
- Big data represents terabytes to petabyte scale data, collected from various sources
- Organizations can derive insights using analytics, or provide forecast for future using Machine Learning, or generate new content using Generative AI
- Big data is often referred to as five V's
	- Volume
		- Amount of data
	- Velocity
		- Speed or the frequency data is being collected or processed
	- Variety
		- Type of data
			- Structured
				- Relational databases (CSV)
			- Semi-Structured
				- JSON, XMLs, Emails
			- Unstructured
				- Media files (audio, video, PDF)
	- Veracity
		- Reliable data
	- Value
		- Worth of data collected

- Big Data needs a distributed processing framework that can parallel process with a cluster of servers
	- Hadoop
	- Spark

Distributed Processing Frameworks for Big Data
- Distributed processing mean that the processing happens in a multi-node cluster instead of a single virtual or physical machine
- Hadoop framework became popular because of its massive parallel processing capabilities on top of fault tolerant nature
- Hadoop clusters consist of a master node of several data nodes that can scale horizontally as the data or processing volume increases
- A common storage layer build on top of all the data node disks is called Hadoop Distributed File System (HDFS)

MapReduce
- Distributed processing framework
- Originally designed to process data concurrently from HDFS by splitting data into 64 MB or 128 MB
	- Map
		- Takes input (key, value), transforms them and writes an other pair an intermediate output
	- Reduce
		- Process that takes (key, value) pair as input, aggregate them and write output as another (key, value) pair
	- Combiner
		- Optional stage that acts as a mini reducer
		- Aggregates records available in a single data node


Spark
- Apache Spark is an open source framework that provides query execution
- APIs
	- Java
	- Scala
	- Python
	- R
- Data is read or written as Resilient Distributed Datasets (RDDs), Dataframes, or Datasets
- 2 Operations
	- Transformations
		- Operations in which you crate a new RDD, Dataframe or Dataset from an existing one after applying transformation rules such as map, filter, join
		- Lazy manner
		- Transformations are only executed when Action operations are invoked
	- Actions
		- Operations that trigger transformations
		- Returns the output to the Spark Driver program
- Spark provides the option to execute in a standalone mode or in a cluster mode
	- A job gets splits into multiple tasks that get executed in different worker nodes
- Each works node may have more than one executed, with a fixed amount of memory and CPU core for execution
- Each worker node has its own cache memory to which you can write small datasets that may be repeatedly used by multiple tasks

Flink
- Apache Flink is an open-source distributed processing framework that is designed for real-time stream processing of high throughput workloads
- Processing in-memory and is stateful while processing both unbounded streaming data or bounded data as a batch
	- Out-of-order event processing
	- Once semantics
	- Back pressure control
- Two components
	- Job manager
	- Task manager

Job Manager
1) Resource Manager
2) Dispatcher
3) Job Master


Task Managers
- Execute the tasks of a dataflow that includes multiple Task Slots
	- Scale horizontally for concurrent execution
