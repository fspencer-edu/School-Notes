# Linear Models

- A linear model is the output of the sum of attribute values, except that weights are applied to each attribute before adding them together
- Regression is the process of predicting a numeric quantity, and regression model is another term for this kind of linear model
- Linear models can be applied to binary classification problems
- A decision boundary separate two classes
- The boundary becomes a high-dimensional plane, or "hyperplane" in the instance space
- The weight were found by a method called least squares linear regression


# Trees

- A "divide-and-conquer" approach leads to a style of representation called a decision tree
- Nodes in a decision tree involve testing a particular attribute
- Leaf nodes give a classification that applied to all instances that reach the left
- If a node is nominal, the number of children is the number of possible values of the attribute
- If the attribute is numeric, the test node determines whether the value is greater or less than
- A three-way split can be used
- A functional tree can have oblique splits as well as linear models at the leaf nodes, which are used for prediction
	- Such nodes are called option nodes
- Decision trees with averages numeric values at the leaves are called regression trees

# Rules

- The antecedent, or precondition, or a rule is a series of test just like the tests at nodes in decision trees
- Consequent, or conclusion, gives the class or classes that apply to instances covered by that rule

## Classification Rules

- The antecedent of the rule includes a condition for every node on the path from the root to that leaf
- The consequent is the class assigned by the leaf
- Decision trees cannot easily express disjunction implied among the different rules in a set
- A replicated subtree problem occurs when a root node results in two subtrees
- A closed-world assumption results in no ambiguity in rule interpretation
	- Disjunctive normal form
- Machine learning algorithms that generate rules invariably produce ordered rule sets in multi-class situations


## Association Rules

- Association rules are not intended to be used together as a set, as classification rules are
- The coverage of an association rule is the number of instances for which it predicts correctly, often called support
- Accuracy, also called confidence, is the number of instances that it predicts correctly, expressed as a proportion of all instances

## More Expressive Rules

- Many machine learning schemes do not consider relations between attributes because there is a considerable cost
- Secondary attributes help define relations
- Logic programs and inductive logic programming used induction theory

# Instance-Based Representation

- The simplest form of learning is plain memorization, or rote learning
	- Instance-based learning
- In instance-based classification, each new instance is compared with existing ones using a distance metric, and the closest is used to assign the class to the new one
	- Nearest-neighbour
	- $k$-nearest-neighbour
- Deriving suitable attribute weights from the training set is a key problem in instance-based learning
- A more complex kind of generalization is to permit rectangular regions to nest one within another

# Clusters

- Some cluster algorithms allow one instance to belong to more than one cluster
- Some algorithms associate instances with clusters probabilistically rather than categorically
- Other algorithms produces hierarchical structure of clusters
	- Dendrograms
- Clustering is often followed by a stage in which a decision tree or rule set is inferred that allocated each instance to the cluster in which it belongs

-  Conflict resolution strategies have been developed for use with rule-based programming systems
