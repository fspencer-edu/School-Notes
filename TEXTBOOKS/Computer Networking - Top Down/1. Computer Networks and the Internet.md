### Packet Switching

- End systems exchange messages with each other
- Messages contain anything the application wants
- Message perform functions or contain data
- The source breaks messages into small chucks known as packets
- Between source and destination, packets are sent through packet switches
	- Routers
	- Link-layer switches
- Packets are transmitted over link at a rate equal to the full transmission rate
- A packet of $L$ bits over a link with transmission rate $R$ bits/sec, has a transmit time of $L/R$ seconds

#### Store-and-Forward Transmission

- Most packet switches use store-and-forward transmission at the inputs
- Packet switch must receive the entire packet before it can transmit the first bit of the packet onto the outbound link
- A router will have many incident links
	- Its job is to attach incoming packets to the corresponding link


<img src="/images/Pasted image 20251211193902.png" alt="image" width="500">

- Source has 3 packets to a destination
	- Each consisting of $L$ bits
- The source transmitted packet 1, which has arrived at the router
- Router puts in buffer
- Then transmits packets when all bits have arrived
- Source begins to transmit at time $0$
- At $L/R$ for the router has received the entire packet
- Therefore, at $2L/R$, the router has transmitted the entire packet

For 3 Packets
- $2L/R$ for first packet to destination, and 2nd packet to router
- $3L/R$, 1st and 2nd at destination, 3rd at router
- Therefore, $4L/R$ for all three packets

General
- $N$ links each of rate $R$
- $N-1$ routers between source and destination

$d_{end-to-end}=NLR$

#### Queuing Delays and Packet Loss

- Each packet switch has multiple links
- For each link, the packet switch has an output buffer/queue
	- Stores packet in router
- In addition to store-and-forward delays, packets suffer output buffer queuing delays
- Delays are variable and depend on level of congestion in the network
- Arriving packet may encounter a full buffer
	- Resulting in packet loss

### Circuit Switching

**2 Approaches to Moving Data Through Links and Switches**
1) Circuit switching

**Circuit Switching**

	- Resources are not reserved
	- A session's messages use the resources on demand, therefore they have to wait

- Packet switching - No reservations
- Circuit switching - reservation

#### Multi-plexing in Circuit-Switched Networks

- A circuit is a link implemented with either frequency-division multiplexing (FDM) or time-division multiplexing (TDM)

**FDM**
- Frequency spectrum of a link is divided among the connections established across the link
- Link dedicates a frequency band to each connection
	- Phone: 4 kHz
- The width of the band is called the bandwidth
- FM radio stations also use FDM to share the frequency spectrums between 88 MHz and 108 MHz

**TDM**
- Time is divided into frames of fixed duration
- Each frame is divided into a fixed number of time slots
- When a connection is established, it dedicates one time slot in every frame

<img src="/images/Pasted image 20251211201816.png" alt="image" width="500">

- Proponents of packet-switching have agued that circuit switching is wasteful because dedicated circuits are idle during silent periods
- Also point out that establishing end-to-end circuits and reserving transmission capacity is complicated and requires complex signalling to coordinate the operation

- Host A to B sends a file 640,000 bits over a circuit-switched network
- All links in use TDM with 24 slots with a bit rate of 1.536 Mbps
- 500 msec to establish end-to-end circuit before A can begin to transmit the file

_How long does it take to send the file?_

$L/R$ = $T_{transmission}$

$T_{setup} = 500$ ms

Circuit rate = $1.536/24 = 64$ kpbs

$T_{transmission} = 640000/64000 = 10$ seconds

Total time = $T_{setup} + T_{transmission} = 0.5 + 10 = 10.5$ seconds

Therefore, the total time to send the file is 10.5 seconds

## Delay, Loss, and Throughput in Packet-Switched Networks

### Overview of Delay in Packet-Switched Networks

- Packet suffers from several types of delays at each node along the path
	- Nodal processing delay
	- Queuing delay
	- Transmission delay
	- Propagation delay
- All these delays accumulate to give total nodal delay

**Types of Delays**
- Packet can be transmitted on a link only if there is no other packets currently being transmitted and there are not other packets preceding it in the queue

![[Pasted image 20251212110645.png]]


**Processing Delay**
- The time required to examine the packet's header and determine where to direct the packet
- Time needed to check for bit-level errors
- Microseconds or less
- Router directs the packets to the queue that precedes the link to router B

**Queuing Delay**
- Waits to be transmitted onto the link
- Order or microseconds to milliseconds

**Transmission Delay ($L/R$)**
- Packets can be transmitted only after all the packets have arrived
- Transmission delay is the amount of time required to push all the packet's bit into the link
- Order or microseconds to milliseconds
- Amount of time required for the router to push out the packet
- Does not use distance

**Propagation Delay ($d/s$)**
- Propagates to router B
- Time to propagate from the beginning of the link to B
- Speed depends on physical medium and range
- Distance between two routers divided by the propagation speed
- Time it takes a bit to propagate from one router to the next
- Does no use packet length or transmission rate

$d_{nodal}=d_{proc}+d_{queue}+d_{trans}+d_{prop}$

### Queuing Delay and Packet Loss


- Queuing delay can vary from packet to packet
- Therefore uses statistical measures
	- Averaging queuing delay
	- Variance of queuing delay
	- Probability that the queuing delay exceed some specific value

$L_a$ = average rate of bits
$R$ = transmission rate

$L_a/R$ = traffic intensity

- If > 1
	- Then the average rate at which bits arrive exceed the rate of transmission from queue
- Therefore, design the system so that the traffic intensity is no greater than 1

- If <= 1
	- Packets arrive periodically
	- First packet as no delay
	- Second packet has queuing delay of $L/S$
	- nth packet has queuing delay of $(n-1)L/R$

- The arrival process to a queue is random
- If traffic intensity is close to zero, then packets rarely have a queuing delay
- If traffic intensity approaches 1, then queue length get exponentially larger

![[Pasted image 20251212112155.png]]

**Packet Loss**
- Queue capacity is finite
- Packet delays do not approach infinity
- If a packet arrives at a full queue, with no storage, a router will drop that packet
- Packet loss will be transmitted but never arriving at the destination
- Node performance is measured in terms of probability of packet loss

### End-to-End Delay

- Total delay from source to destination
- N-1 routers between the source and destination

$d_{end-end} = N(d_{proc}+d_{trans}+d_{prop})$

**Traceroute**
- Traceroute program is used to run internet host
- Source can reconstruct the route taken by packets flowing from source to destination
- There are 9 routers between the source and destination
- First of the 3 trials, the round-trip delay was 1.03 msec
- Subsequent trials were 0.48 and 0.48 msec

**End System, Application, and Other Delays**
- End systems want to transmit a packet into a shared medium (WiFi or cable) may purposefully delay transmission in its protocol
- Media packetization delay
	- Present in Voice-over-IP (VoIP)

## Protocol Layers and Their Service Models

### Layered Architecture

- Layered architecture allows for a well-defined, and specific parts of a large and complex system
	- Modularity
- Ability to change the implementation of a service without affecting other components

**Protocol Layering**
- Each protocol belongs to one of the layers
- Service model of a layer
	- Performs certain actions within that layer
	- Uses the services of the layer directly below it
- Implemented in software, hardware, of both
- Application layer protocols
	- HTTP
	- SMPT
- One drawback of layering is that on layer may duplicate lower-layer functionality
- Protocols stacks provide error recovery on both a per-link basis and end-to-end basis
- Functionality at one layer may violate the another layer
- Layers are called the protocol stack
- Internet consists of 5 layers

![[Pasted image 20251212114316.png]]


**Application Layer**
- Network applications and their application-layer protocol reside
- HTTP
- SMPT
- FTP
- Domain name system (DNS)
- Packet information at this layer is called a message

**Transport Layer**
- Transport messages between application endpoints
- TCP
	- Connect-oriented service
	- Guaranteed delivery of application-layer messages to destination
	- Breaks long messages into shorter segments and provides congestion-control
- UDP
	- Connectionless service to its application
	- No flow control
	- No congestion control
- Transport layer packet is referred as a segment

**Network Layer**
- Internet's network layer move packets known as datagrams
- Internet transport-layer protocol (TCP or UDP) in source host passes a transport-layer segment and a destination address to the network layer
- Network layer then provides the service delivering the segment to the transport layer in destination host
- IP protocol
	- Defines the fields in the datagram
	- End systems and routers act on these fields
- Only one IP protocol
- Contains routing protocols that determine the routes that datagrams take between sources and destinations

**Link Layer**
- Network layer relies on link layer
- At each node, the network layer passes the datagram down to the link layer, which delivers the datagram to the next node
- Link layer passes the datagram up to the network layer
- Specific link-layer protocols
	- Reliable delivery is based on service of TCP
	- Ethernet
	- WiFi
	- Cable access network's DOCSIS protocol
- Refer to link-layer packets as frames

**Physical Layer**
- Move entire frame from one network element to an adjacent network element
- Move the individual bits within the frame from one node to the next
- Dependent on medium of link

**The OSI Model**
- International Organization for Standardization (ISO) proposed that computer networks be organized around 7 layers
- Open Systems Interconnection (OSI) model
- Late 1970s
- Used in network education
- Adds
	- Presentation layer
	- Session layer

**Presentation Layer**
- Provide service that allow communicating applications to interpret the meaning of data exchange
- Data compression
- Data encryption
- Data description

**Session Layer**
- Provides delimiting and synchronization of data exchange
- Build a checkpointing and recovery scheme

### Encapsulation

## History of Computer Networking and the Internet

### The Development of Packet Switching (1961-1972)

- Networking and internet trace back to the early 1960s
- Telephone network uses circuit switching to transmit information
- The first packet-switched computer network and a direct ancestor to today's internet was the ARPAnet (Advanced Research Project Agency)
- First host-to-host protocol between ARPAnet end systems, known as the network control protocol (NCP) in 1972

### Proprietary Networks and Inter-networking (1972-1980)

- ARPAnet was a single, closed network
- ALOHANet
- DARPA's packet satellite
- Packet radio networks
- Early versions of TCP combined reliable in-sequence delivery of data via end-system retransmission with forwarding functions
