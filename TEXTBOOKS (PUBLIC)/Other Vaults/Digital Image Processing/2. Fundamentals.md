# Digital Image Representation

- f(x, y) are the spatial coordinates
- f is the amplitude called the intensity
- Gray level is used to refer to the intensity of monochrome images
- Colour images are formed by a combination of 2D images
- RGB colour system
	- Red
	- Green
	- Blue
- Any image may be continuous with respect to the x and y coordinates, and amplitude
- Digitizing the coordinate values is called sampling
- Digitizing the amplitude values is called quantization
- When x, y, and f are all finite, discrete quantities, the image is a digital image

## Coordinate Conventions

- The result of sampling and quantization is a matrix of real numbers
- M - rows, and N - columns
- Image is of size M x N
- The image origin is defined at (0, 0)
- The notation (0, 1) is used to signify the second sample along the first row

- (r, c) indicates rows and columns
- Origin starts at (1, 1) in integer increments
- IPT refers to the coordinates as pixel coordinates
- Toolbox also uses spatial coordinates
	- x refers to columns
	- y refers to rows

# Converting between Data Classes and Image Types

- Converting between data classes and images types is a frequent operation

## Converting between Data Classes

```c
B = data_class_name(A)
B = double(A)
```
- MATLAB expects operands to be in double-precision, floating-point
- Values outside a range are brought to the closest max and min values
- Proper scaling of a double array is necessary before converting to `uint8`

## Converting Between Image Classes and Types

```c
f = 
	-0.5 0.5
	-75  1.5
g = im2uint8(f)
	0 128
	191 255
```
- `im2uint8()` multiples all values by 255
- Rounding to the nearest integer
- Converting an arbitary array of class `double` to an array of class `double` scaled to the range [0, 1] uses

```c
g = mat2gray(A, [Amin, Amax])
g = mat2gray(A)
```
- `g` has values in the range 0 (black) to 1 (white)

- Function `im2double` converts an input to class double

```c
h = uint8([25 50; 128 200]);
g = im2double(g);
	0.0980 0.1961
	0.4706
	0.7843
```
- Divide each value of the input array by 255

```c
g = im2bw(f, T)
```
- Converts binary to intensity image types
- Uses default value 0.5 for T
- If an `uint8` images, `im2bw` divides all pixels by 255 and applies a specified threshold
- A logical array can be converted to a numerical array using the 4 functions in the first column of the below table

- To convert the double image to binary
```c
f = [1 2; 3 4]
g = mat2gray(f)
	0 0.3333
	0.6667 1.0000
// Convert it to binary using a threshold
gb = im2bw(g, 0.6)
	0 0
	1 1
// Generate binary directly with relational operators
gb = f > 2
	0 0
	1 1
gbv = islogical(gb)
	1
// convert logical array to double
gbd = im2double(gb)
	0 0
	1 1
```
