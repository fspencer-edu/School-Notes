- OpenGL is an application program interface (API)

Major Operation in OpenGL Application
- Specify the data for constructing shapes
- Execute various shaders
- Convert the mathematical description
- Execute a fragment shader
- Perform additional pre-fragment operations

- OpenGL is implemented as a client-server system

Basic Structure
- Initialize the state
- Specify those objects to be rendered

- Rendering is the process by which a computer creates an image from models
- OpenGL is a rasterization-based system
- Models or objects are constructed from geometric primitives
	- Points
	- Lines
	- Triangles
- Shaders are special functions that the graphics hardware executes
- In OpenGL there are 4 shader stages
	- Vertex shader
	- Fragment shader
- A pixel is the smallest visible element on your display
- Pixels are stored in a frame buffer, which is a chuck of memory that the graphics hardware manages

Example Code
- Include header files and declare global variables
- The `init()` routine is used to set up data
- `display()` does the rendering
- `glClear()` clears the window
- `main()` does the heavy lifting of creating a window, calling, `init()`, and entering into the event loop
- OpenGL program across the different operating and window systems
	- GLUT
	- GLEW

# Rendering Pipeline

- A rendering pipeline is a sequence of processing stages for converting the data your application provides to OpenGL into a final rendered image

![[Pasted image 20250910122303.png]]

- Geometric data
- Vertex shading
- Tessellation shading
- Geometry shading

- Only vertex and fragments shaders must be included
- Tessellation and geometry shaders are optional


## Preparing to Send Data

- Data is stored in buffer objects, which are chucks of memory managed by the OpenGL server
- `glBufferData()`

## Sending Data to OpenGL

- `glDrawArrays()`
- A vertex is a bundle of data values that are processed together


## Vertex Shading

- Vertex shaders may be very simple (pass-through shader) to complex that perform many computation (transformation matrices)

## Tessellation Shading

- Tessellation shader stage will continue processing those data
- Tessellation uses patchs to describe an object's shape, and allows relatively simple collection of patch geometry to be tessellated to increase the number of geometric primitives

## Geometry Shading

- Geometry shading allows additional processing on individual geometric primitives, including creating new ones, before rasterization

## Primitive Assembly

- The primitive assembly stage organizes the vertices into their associated geometric primitives in preparation for clipping and rasterization

## Clipping

- Vertices will be outside the viewport and cause the primitive associated with that vertex to be modified

## Rasterization

- Primitives are sent to the rasterizer for fragment generation

## Fragment Shading

- Shader determine the fragment's final colour, and depth value
- Employ texture mapping
- Fragment discard

## Per-Fragment Operations

- Depth testing
- Stencil testing

- Pixel data is stored in texture nap for use with texture mapping, which allows any texture stage to look up data values from one or more texture map

## OpenGL Initialization

```c
void
init(void)
{
	glGenVertexArrays(NumVAOs, VAOs);
	glBindVertexArray(VAOs[Triangles]);

	GLfloat vertices[NumVertices][2] = {
		{ -0.90, -0.90 }, // Triangle 1
		{ 0.85, -0.90 },
		{ -0.90, 0.85 },
		{ 0.90, -0.85 }, // Triangle 2
		{ 0.90, 0.90 },
		{ -0.85, 0.90 }
	};

	glGenBuffer(NumBuffers, Buffers);
	glBindBuffer(GL_ARRAY_BUFFER, Buffers[ArrayBuffer]);
	glBufferData(LG_ARRAY_BUFFER, sizeof(vertices),
				vertices, GL_STATIC_DRAW);

	ShaderInfo shaders[] = {
		{ GL_VERTEX_SHADER, "triangles.vert" },
		{ GL_FRAGMENT_SHADER, "triangles.frag" },
		{ GL_NONE, NULL }
	};

	GLuint program = LoadShaders(shaders);
	glUseProgram(program);
	glVertexAttribPointer(vPosition, 2, GL_FLOAT,
						GL_FALSE, 0, BUFFER_OFFSET(0));
	glEnableVertexAttribArray(vPosition);
}
```

**Initializing Our Vertex-Array Objects**

`glGenVertexArrays(GLsizei n, GLuint *arrays)`
- Allocates a vertex-array object
- `NumVAOs` are specified in the global variable

`glBindVertexArray(GLuint array)`
- Creates and assigns a new vertex-array object
- When an object binds for the first time, `glBind*()` is called for a particular object name
- Created object will be initialized to its default state

`glDeleteVertexArrays(GLsizei n, GLuint *arrays)`
- Deletes a vertex-array object
- Bindings for that vertex array become zero

`glIsVertexArray(GLuint array)`
- Returns GL_TRUE if array is the name of a vertex-array object that was previously generated with `glGenVertexArrays()`, but has not been deleted


**Allocating Vertex-Buffer Objects**

- Vertex-array object holds various data related to a collection of vertices
- Data is stored in buffer objects and managed by currently bound vertex-array object

`glGenBuffers(GLsizei n, GLuint *buffers)`
- Returns n currently unused names for buffer objects in the array

`glBindBuffer(GLenum target, GLuint buffer)`
- Brings buffers into existence
- When binding to a buffer value of zero, OpenGL stops using buffer objects for that target
- 8 types of buffer objects
	1) GL_ARRAY_BUFFER
	2) GL_ELEMENT_ARRAY_BUFFER
	3) GL_PIXEL_PACK_BUFFER
	4) GL_PIXEL_UNPACK_BUFFER
	5) GL_COPY_READ_BUFFER
	6) GL_COPY_WRITE_BUFFER
	7) GL_TRANSFORM_FEEDBACK_BUFFER, or GL_UNIFORM_BUFFER


`glDeleteBuffers(GLsizei n, const GLuint *buffers)`
- Delete buffer objects
- All bindings to that object are reset to the default buffer object

`glIsBuffer(GLuint buffer)`
- Returns GL_TRUE if buffer is the name of the buffer object that has been bound, and not deleted


**Loading Data into Buffer Object**

`glBufferData(GLenum target, GLsizeiptr size, const GLvoid *data, GLenum usage)`
- Transfers the vertex data from objects into the buffer object
	- Allocates storage for holding the vertex data
	- Copies the data from arrays in the application server

- Range of coordinates are known as normalized-device coordinates (NDC)
	- [-1, 1]


**Initializing Our Vertex and Fragment Shaders**

- A vertex shader and a fragment shader must be provided
- A shader is a small function in OpenGL shading Language, a special language similar to C++ for constructing OpenGL shaders
- GLSL (OpenGL Shading Language)

Vertex Shader to `triangles.cpp`: triangles.vert

```C++
#version 430 core
layout(location = 0) in vec4 vPosition;
void
main()
{
	gl_Position = vPosition;
}
```

- Every shader should have a #version line
- Shader variables are a shader's connection to the outside world
- `vPosition` holds the vertex's positional information
- GLSL assigns `vec4` to a 4-component vector of floating-point values
	- Default for `vec4` is (0, 0, 0, 1), or (x, y) = (0, 1)
- `in` specifies which direction data flows into the shader
- `layout(location = 0)` is the layout qualifier, and provides meta-data for variable declaration
- Every shader has a `main()` routine
- This shader copies the input vertex position to the special vertex shader output
- A fragment shader is needed with the vertex shader

Fragment Shader for triangles.cpp: triangles.fram

```C++
#version 430 core
out vec4 fColor;
void
main()
{
	fColor = vec4(0.0, 0.0, 0.0, 1.0);
}
```

- `fColor` declares a variable of `vec4`
- `out` qualifier, output values through `fColor`
- Each fragment is assigned this vector of 4 values
- RGBA = red, green, blue, alpha
	- Alpha measures translucency

## Our First OpenGL Rendering

```c
void
display(void)
{
	glClear(GL_COLOR_BUFFER_BIT);
	glBindVertexArray(VAOs[Triangles]);
	glDrawArrays(GL_TRIANGLES, 0, NumVertices);
	glFlush();
}
```

`glClear(GLbitfield mask)`
- Clears the specified buffers to their current clearing values
- `mask` is bitwise logical OR combination of the value
- OpenGL default clearing colour is black


`glClearColor(GLclampf red, GLclampf green, GLclampf blue, GLclampf alpha`
- Sets the current clear colour for use in clearing colour buffers in RGBA mode
- Retains any state change

**Drawing with OpenGL**

`glDrawArrays(GLenum mode, GLint first, GLsizei count)`
- Constructs a sequence of geometric primitives using the elements from the currently bound vertex array starting at first and ending at `first + count - 1`

`glFlush()`
- Requests that any pending OpenGL calls are flushed to the OpenGL server and processed
- Requests all pending commands be sent to server, and returns

**Advanced**

`glFinish(void)`
- Forces the completion of all pending OpenGL commands and waits for their completion

**Enabling and Disabling Operations in OpenGL**


`glEnable(GLenum capability)`
`glDisable(GLenum capability)`
- Turns on and off a capability

`glIsEnabled(GLenum capability)`
- Returns GL_TRUE or GL_FALSE, depending on whether or not the queried capability is currently activated
