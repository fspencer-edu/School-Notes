- The geometry shader sits logically before primitive assembly and fragment shading
- Receives as input complete primitives as a collection of vertices
- Geometry shader can output a variable amount of data
- Producing a different primitive type as its output than it accepts on its input

# Geometry Amplification

- Amplification
	- Shader that produces more primitives on its output than it accepts on its input

**Fur Rendering Using a Geometry Shader**

- Fur-rendering effect
- Hair or fur on a surface is modelled as a volume that is rendered using slices, and the geometry shader is used to generate those slices
- The input to the geometry shader is the triangles forming the underlying mesh
- The geometry shader produces the fur shells by displacing the incoming vertices along their normals and producing multiple copies of the incoming geometry

_Fur Rendering Geometry Shader_
```c
#version 330 core

layout (triangles) in;
layout (tiangle_strip, max_vertices = 120) out;

uniform mat4 model_matrix;
uniform mat4 projection_matrix;

uniform in fur_layers = 30;
uniform float fur_depth = 5.0;

in VS_GS_VERTEX
{
	vec3 normal;
	vec3 tex_coord;
} vertex_in[];

out GS_FS_VERTEX
{
	vec3 normal;
	vec2 tex_coord;
	flat float fur_strength;
} vertex_out;

void main()
{
	int i, layer;
	float disp_detla = 1.0 / float(fur_layers);
	float d = 0.0;

	for (layer = 0; lauer < fur_layers; layer++)
	{
		for (i = 0; i < gl_in.length(); i++){
			vec3 n = vertex_in[i].normal;
			vertex_out.normal = n;
			vertex_out.tex_coord = vertex_in[i].tex_coord;
			vertex_out.fur_stength = 1.0 - d;
			position = gl_in[i].gl_Position + vec4(n * d * fur_depth, 0.0);
			gl_Position = projection_matrix * (model_matrix * position);
			EmitVertex();
		}
		d += disp_detla;
		EndPrimitive();
	}
}
```

# Advanced Transform Feedback

- The output of the vertex shader is captured and recorded into one or more buffer objects
- Multiple vertex streams as outputs from the geometry shader
- Use data generated by a geometry shader and stored into a transform feedback buffer in subsequent draw commands without the CPU

# Multiple Output Streams

- Output streams are declared using the `stream` layout qualifier
- Applied globally, to an interface block, or to a single output declaration
- When a stream number is given a global scope, all subsequently declared geometry shader outputs become members of that stream
- Default stream is zero

_Global Layout Qualifiers Used to Specify a Stream Map_

**Combining Multiple Streams and DrawTransformFeedback**

- Use the geometry shader to sort "left-facing" and "right-facing" polygons

_Simple Vertex Shader for Geometry Sorting_
```c
#version 330 core

uniform mat4 model_matrix;

layout (location = 0) in vec4 position;
layout (location = 1)) in vec3 normal;

out vec3 vs_normal;

void main()
{
	vs_normal = (model_matrix * vec4(normal, 0.0)).xyz;
	gl_Position = model_matrix * position;
}
```

_Geometry Shader for Geometry Sorting_
```c
#version 330 core

layout (triangles) in;
layout (points, max_vertices = 3) out;

uniform mat4 projection_matrix;

in vec3 vs_normal[];

layout (stream = 0) out vec4 lf_position;
layout (stream = 0) out vec3 lf_normal;

layout (stream = 0) out vec4 rf_position;
layout (stream = 0) out vec3 rf_normal;

void main()
{
	vec4 A = gl_in[0].gl_Position;
	vec4 B = gl_in[1].gl_Position;
	vec4 C = gl_in[2].gl_Position;
	vec3 AB = (B - A).xyz;
	vec3 AC = (C - A).xyz;
	vev3 face_normal = cross(AB, AC);
	int i;

	if (face_normal.x < 0.0)
	{
		for (i = 0; i < gl_in.length(); i++)
		{
			lf_position = projection_matrix * (
						gl_in[i].gl_Position - vec4(30.0, 0.0, 0.0, 0.0));
			lf_normal = vs_normal[i];
			EmitStreamVertex(0);
		}
		EndStreamPrimitive(0);
	}
	else
	{
		for (i = 0; i < gl_in.length(); i++)
		{
			rf_position = projection_matrix *
						gl_in[i].gl_Position - vec4(30.0, 0.0, 0.0, 0.0));
			rf_normal = vs_normal[i];
			EmitStreamVertex(1);
		}
		EndStreamPrimitive(1);
	}
}
```

- When rendering the sorting pass, we will not be rasterizing any polygons

# Geometry Shader Instancing

- Another types of instancing is known as geometry shader instancing, is available that only runs the geometry shader and subsequent stages (rasterization and fragment shading) multiple time, rather than the whole pipeline

```c
layout (triangles, invocations = 4) in;
```
- Geometry shader will be invoked 4 times for each input primitive
- Most as much work as possible from the geometry shader to the vertex shader
- Work is performed only once and then shared across all geometry shader invocation
- All OpenGL implementations must support at least 32 invocations for instanced geometry shaders, but the count may be higher
- Nest 3 levels of geometry in a single draw

_Geometry Amplification Using Nested Instancing_
```c

	{
		for each primitive produced by the geometry shader
		{
			render primtive
		}
	}
}
```

_Geometry Shader for Rendering into an Array Texture_
```c
#version 330 core
layout (triangles) in;
layout (triangles_strip, max_vertices=128) outl

in VS_GS_VERTEX
{
	gec4 color;
	vec3 normal;
} vertex_in[];

out GS_FS_VERTEX
{
	vec4 color;
	vec3 normal;
} vertex_out;

uniform mat4 projection_matrix;
uniform int output_slices;

void main()
{
	int i, j;
	mat4 slice_matrix;
	float alpha = 0.0;
	float delta = float(output_slices - 1) * 0.5 / 3.1415927;

	for (j = 0; j < output_slices; j++)
	{
		float s = sin(alpha);
		float c = cos(alpha);
		slice_matrix = mat4(vec4(c, 0.0, -s, 0.0),
						vec4(0.0, 1.0, 0.0, 0.0),
						vec4(s, 0.0, c, 0.0),
						vec4(0.0, 0.0, 0.0, 1.0));
		slice_matrix = slice_matrix * projection_matrix;
		for (i = 0; i < gl_in.length(); ++i)
		{
			gl_Layer = j;
			gl_Position = slice_matrix * gl)in[i].gl_Position;
			vertex_out.color = vertex_in[i].color;
			vertex_out.normal = vertex_in[i].normal;
			EmitVertex();
		}
		EndPrimitive();
	}
}
```

- The above example amplified the incoming geometry and renders a complete copy of it into each layer of the layered colour attachment of the current framebuffer
- Impostors are views of real geometry rendered into textures and then used in place of that geometry
