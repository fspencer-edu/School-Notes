# Homogenous Coordinates

Moving from 3D Cartesian to 4-component homogeneous
1) Apply perspective
2) Translate the model using a linear transform



**Advanced: What Are Homogeneous Coordinates?**

- 3D data can be scaled and rotated with linear transformations of 3-components vectors
- Translating 3D Cartesian coordinates cannot be done with multiplication
	- Requires an affine transformation, which is not linear
- Homogeneous coordinates act as directions instead of locations
	- Scaling a direction leaves it pointing in the same direction


- Move to homogeneous coordinates by adding a fourth w component of 1.0

$(2.0, 3.0, 5.0) → (2.0, 3.0, 5.0, 1.0)$

- To go back to Cartesian coordinates, divide all component by the fourth component, and then drop it

$(4.0, 6.0, 10.0, 2.0) divide by w → (2.0, 3.0, 5.0, 1.0) drop w → (2.0, 3.0, 5.0)$


- Making w larger can make coordinates appear further away
- OpenGL will transform homogeneous coordinates back to 3D by dividing their first 3-component by the last component

# Transformating Normals

- Normals are required to be normalized

- Normal vectors are 3-component vectors

## Advanced: User Clipping

- OpenGL automatically clips geometry against the near and far plans and viewport
- Use clipping refers to adding additional clip planes at arbitrary orientations, intersecting geometry, such that the display sees the geometry on one side of the plane, but not the other side
- `gl_ClipDistance` controls where vertices are in relation to a plane
- Normal interpolation assigns distance to the fragments between vertices

_Simple Use of `gl_ClipDistance)

```c
#version 330 core

unifrom vec4 Plane; // Ay + Bx + Cz + D = 0
in vec4 Vertex;

float gl_ClipDistance[1];

void main()
{
	gl_ClipDistance[0] = dot(Vertex, Plane)
}
```
- The distance of 0 means the vertex in on the plane
- A positive distance means the vertex is inside of the clip plane
- A negative distance means the point is outside
- Each element is set up to represent one plane
- To include enabled planed
	- `glEnable(GL_CLIP_PLANE0)`


# Transform Feedback

- Transform feedback captures vertices as they are assembled into primitives and allows some or all of their attributes to be recorded into buffer objects
- Each time a vertex passes through primitive assembly, attributes that have been marked for capture are recorded into one or more attributes

## Transform Feedback Objects

- The state required to represent transform feedback is encapsulated into a transform feedback object
	- Buffer objets
	- Counters
	- State
- A transform feedback object is created by reserving a transform feedback object name and then binding it to the transform feedback object binding point on the current context

`void glGenTransfromFeedbacks(GLsizei n, GLuint * ids)`
- Reserves n names for transform feedback objects and places the reserved names in the array ids

- Parameter `n` specifies how many transform feedback object names are to be reserved
- `ids` specifies the address of an array

`void glBindTransformFeedback(GLenum target, GLuint id)`
- Binds a transform feedback object specified by id to the target

`GLboolean glISTransformFeedback(GLenum id)`
- Returns GL_TRUE if id is the name of an existing transform feedback object

- Becomes convenient to encapsulate the state of transform feedback into transform feedback objects
- Create and bind a transform feedback object even if you intend to use only one

`void glDeleteTransfromFeedbacks(GLsizei n, const GLuint ids)`
- Deletes n transform objects whose names are stored in the array ids
- Deletion does not occurs until transform feedback is ended

## Transform Feedback Buffers

- Transform feedback objects are primarily responsible for managing the state representing capture of vertices into buffer objects
- To bind an entire buffer object to one of the transform feedback buffer binding points, call:

`void glBindBufferBase(GLenum target, GLuint index, GLuint buffer)`
- Binds the buffer object with the name buffer to the indexed binding point on target at the index
- All OpenGL implementations must support at least 64 transform feedback buffer binding points

`void glBindBufferRange(GLenum target, GLuint index, GLuint buffer, GLintptr offset, GLsizeptr size)`
- Binds a range of a buffer object to the indexed buffer binding point on target specified by index


_Example Initialization of a Transform Feedback Buffer_
```c
// Generate the name of a buffer object
GLuint buffer;
glGenBuffer(1, &buffer);

// Bind and create
glBindBuffer(GL_TRANSFORM_FEEDBACK_BUFFER, buffer);

// Allocate 1MB of space
glBufferData(GL_TRANSFORM_FEEDBACK_BUFFER, 1024 * 1024, NULL, GL_DYNAMIC_COPY);

// Bind to indexed buffer binding points
glBindBufferRage(GL_TRANSFORM_FEEDBACK_BUFFER, 0, buffer, 0, 512 * 1024);

glBindBufferRange(GL_TRANSFORM_FEEDBACK_BUFFER, 1, buffer, 512 * 1024, 512 * 1024);
```

- `glBindBufferRange()` is called twice
	- Bind the first half of the buffer to the first binding point
	- Bind the second half
- Called only after the object exists

## Starting and Stopping Transform Feedback

- Transform feedback can be started, stopped, and paused

`void glBeginTransformFeedback(GLenum primitiveMode)`
- Sets the primitive type expected to the recorded by transform feedback
- Starts on the currently bound transform feedback object
- Must match primitive type expected to arrive at primitive assembly

<img src="/images/Pasted image 20250922144259.png" alt="image" width="500">

- Once transform feedback is started, it is considered to be active

Changing State when pauseed
- The currently bound transform feedback object may not be changed
- Possible to bind different buffers to the `GL_TRANSFORM_FEEDBACK_BUFFER` binding points
- Current program object cannot be changed

`void glPauseTransformFeedback(void)`
- Pauses the recording of varyings in transform feedback mode

`void glResumeTransformFeedback(void)`
- Resumes transform feedback that has previously been paused

`void glEndTransformFeedback(void)`
- Completes the recoding of varyings in transform feedback mode


## Transform Feedback Example - Particle System

- Application uses transform feedback in 2 ways
	- Capture geometry passed through the OpenGL pipeline
	- Vertex shader to perform collision detection between particles and rendered geometry

<img src="/images/Pasted image 20250922144825.png" alt="image" width="500">

- Vertex shader is used to transform object space geometry into both world space and eye space for rendering
- World space are captured into a buffer using transform feedback
- Eye space is passed through to the rasterizer
- Buffer containing the world space is attacked to a texture buffer object (TBO) to be accessed in the vertex shader
- Particle position and velocity vectors are stored in pairs of buffers
- Two buffers are used so that data can be double-buffered
- Each vertex in the buffer represents a single particle in the system

_Vertex Shader Used in Geometry Pass of Particle System Simulator_
