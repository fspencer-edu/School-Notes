# Basic Color Theory

- In the physical world, light is composed of photons
	- Tiny particles traveling along a straight path each with their own "colour" represented by their wavelength
	- 390 nm to 720 nm
- A set of colours a device can display is represented as its gamut
- In addition to RGB, OpenGL also supports the sRGB colour space
- HSV (Hue-Saturation-Value)
- CMYK (Cyan-Magenta-Yellow-Black)
- Computer framebuffers can only represent a comparatively small number of discrete values
- Each component's intensity is stored using a certain number of bits (bit depth)
- Sum of each component's bit depth determines the colour buffer's depth
	- 8 bits for each R, G, and B, or (255, 255, 255)
	- 24-bit deep colour buffer
	- Displaying $2^{24}$ unique colours

# Buffers and Their Uses

- The framebuffer is composed of a rectangular array of pixels, each capable of displaying a tiny square of colour at the point in the image
- After the rasterization stage, the data are not pixels yet, just fragments
- Each fragment has coordinate data that corresponds to a pixel, colour, and depth values
- A pixel (x, y), fills the region bounded by 
	- x on the left, x + 1 on the right, 
	- y on the bottom, and y + 1 on the top

- Pixel may have many colour buffers associated with it, which are called renderbuffers
- Use buffers to perform tasks as hidden-surface removal, stencilling, dynamic texture generation

Type of Buffers
- Colour buffers
- Depth buffer
- Stencil buffer

- Application starts with the default framebuffer
	- Contains a double-buffered, colour buffer

**Color Buffers**

- The "main" colour buffer of the default framebuffer is associated with the window of the screen
- The pixels in a colour buffer may store a single colour per pixel, or may logically divide the pixel into subpixels, which enables an antialiasing technique called multi-sampling
- Double buffering is done by making the main colour buffer have two parts
	- Front buffer
	- Back buffer
- Swap the buffers with, `glutSwapBuffers()`
- OpenGL implementation might support stereoscopic viewing

**Depth Buffer**

- The depth buffer stored a depth value for each pixel, and is used for determining the visibility of objects in 3D space
- Measured in terms of distance to the eye
- Larger depth-buffer values are overwritten by pixels with smaller values
- Also called the z-buffer

**Stencil Buffer**

- Stencil buffer is used to restrict drawing to certain portions of the screen

# Color and OpenGL

Fragment shader assigns a fragment's colour
- The fragment shader can generate the fragment's colour without using external data
- Additional colour data is provided with each input vertex
- Supplemental data can be provided to the fragment shader and used in a computation that generates a colour
- External data, like a digital image, can be referenced in a fragment shader
	- Texture mapping

# Color Representation and OpenGL

- OpenGL internally represents a colour component as a floating-point value and maintains its precision until that value is stored in the framebuffer
	- [0.0, 1.0] - normalized value
- OpenGL will convert the values from the input data type into the suitable normalized-value range

init(void)
{
	glGenVertexArrays(NumVAO, VAOs);
	glBindVertexArray(VAOs[Triangles]);

	struct VertexData {
		GLubyte color[4];
		GLfloat position[4];
	};

	VertexData vertices[NumVertices] = {
		{{ 255, 0, 0, 255 }, { -0.90, -0.90 }}, // Triangle 1
		{{ 0, 255, 0, 255 }, { 0.85, -0.90 }},
		{{ 0, 0, 255, 255 }, { -0.90, 0.85 }},
		{{ 10, 10, 10, 255 }, { 0.90, -0.85 }}, // Triangle 2
		{{ 100, 100, 100, 255 }, { 0.90, 0.90 }},
		{{ 255, 255, 255, 255 }, { -0.85, 0.90 }}
	};

		glGenBuffers(NumBuffers, Buffers);
		glBindBuffer(GL_ARRAY_BUFFER, Buffers[ArrayBuffer]);
		glBufferData(GL_ARRAY_BUFFER, sizeof(vertices), vertices, GL_STATIC_DRAW);

	ShaderInfo shaders[] = {
		{ GL_VERTEX_SHADER, "gouraud.vert" },
		{ GL_FRAGMENT_SHADER, "gouraud.frag" },
		{ GL_NONE, NULL }
	};

	GLuint program = LoadShaders(shaders);
	glUseProgram(program);

	glVertexAttribPointer(vColor, 4, GL_UNSIGNED_BYTE, GL_TRUE, sizeof(VertexData), BUFFER_OFFSET(0));

	glVertexAttribPointer(vPosiition, 2, GL_FLOAT, GL_FLASE, sizeof(VertexData), BUFFER_OFFSET(sizeof(vertices[0].color)));

	glEnableVertexAttribArray(vColor);
	glEnableVertexAttribArray(vPosition);
}

# Rasterization

- Between vertex shading stages (vertex, tessellation, and geometry shading) and fragment shading, is the rasterizer
- Determines which screen locations are converted by a particular piece of geometry
- Linearly interpolates the data values for each varying variable in the fragment shader
	- Gouraud shading
	- Surface normal can also be interpolated
	- Texture coordinates
- Rasterization and interpolation is platform-dependent

# Multisampling

- Multisamping is a technique for smoothing the edges of geometric primitives, known as antialiasing
- Uses multiple samples, which are like mini-pixels, to store colour, depth, and stencil values
- All samples are resolved to determine the final pixel's colour

	// Draw a sphere

# Depth Test

**Polygon Offset**

- The highlighting lines may fade in and out of the coincident polygons, called stitching
- Eliminated by using offset, which adds an appropriate offset to force coincident z-values apart, separating a polygon edge from its highlighting line
- Polygon offset is also useful for applying decals to surface by rendering images with hidden-line removal

`void glPolygonOffset(GLfloat factor, GLfloat units)`
- When enabled, the depth value of each fragment is modified by adding a calculated offset before the depth test is performs

Offset value

$offset = m · factor + r · units$

- Add a positive offset to the solid object
- Or add a negative offset to the wireframe

- The depth slope is the z-value divided by the change in either the x- or y- coordinates traversed by the polygon

# Occlusion Query

**Advanced**

- The depth buffer determines visibility on a per-pixel basis
- Occlusion queriess enable you to determine if a representative set of geometry will be visible after depth testing

Occlusion Queries
1. Generate a query id for each occlusion query that you need
2. Specify the start of an occlusion query by calling `glBeginQuery()`
3. Render the geometry for the occlusion test
4. Specify that you've completed the occlusion query by calling `glEndQuery()`
5. Retrieve the number of, or if any samples passed the depth tests

**Generating Query Objects**

`void glGenQueries(GLsizei n, GLuint *ids)`
- Returns n currently unused names for occlusion query objects in the array ids

`GLboolean glIsQuery(GLuint id)`
- Return GL_TRUE, if id is the name of an occlusion query object


**Initiating an Occlusion Query Test**


_Rendering Geometry with Occlusion Query: occquery.c_
```c

glDrawArrays(GL_TRIANGLES, 0, 3);
glEndQuery(GL_SAMPLES_PASSED);
```

`void glBeginQuery(GLenum target, GLuint id)`
- Specifies that start of an occlusion query operation

`void glEndQuery(GLenum target)`
- Ends an occlusion query

**Determining the Results of an Occlusion Query**

- After rendering the geometry for the occlusion query, retrieve the results

`void glGetQueryObjectiv(GLenum id, GLenum pname, GLint *params)`
`void glGetQueryObjectiv(GLenum id, GLenum pname, GLuint *params)`
- Queries the state of an occlusion query object
