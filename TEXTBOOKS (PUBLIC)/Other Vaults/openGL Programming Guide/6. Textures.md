# Texture Mapping
- Texture mapping allows you to loop up values, like colours, from a shader in a special type of table
- A texture map is an image captured by a camera or painted
- OpenGL supports 2D and 3D textures, cube textures, and buffer textures
- Array textures are also supported, which are treated as a set of slices of similar dimension and format, wrapped in a single texture object
- Textures are composed of texels, which contain colour values

1) Create a texture object and load texel data into it
2) Include texture coordinates with your vertices
3) Associate a texture sampler with each texture map you intend to use in shader
4) Retrieve the texel values through the texture sampler from your shader

# Basic Texture Types

- Texels and many images may be stacked one top of another to form what is know as a mipmap pyramid
- Textures contain arrays of 1D or 2D slices known as array textures, and each element of the array is a slice
- A cube map is a special case of an array texture that has multiple of six slices
- A single cube-map texture has 6 faces, whereas a cub-map array represents an array of cube-map textures, having an integer multiple of six faces
- Multisampling is a term that refers to an implementation of antialiasing where each texel (or pixel) is assigned multiple independent colours and those colours may be merged together later in the rendering process
- Textures are bound to the context via texture units, which are represented as binding points
- A texture can access shaders using sampler variables

- Buffer textures make is possible to access vertex data from any shader stage without needing to copy it to a texture image

# Cube-Map Textures

- Cube-map textures are a special type of texture, useful for environment mapping, that takes a set of images and treats them as the faces of a cube
- 6 faces of a cube must be square and of the same size
- Cube maps are ideal for representing surrounding environments, lighting, and reflection effects, and can also be used to wrap complex object with textures

_Initializing a Cube-Map Texture_
```c
GLuint tex;

extern const GLvoid* texture_data[6];

// Generate a texture object
glGenTextures(1, &tex);
glBindTexture(GL_TEXTURE_CUBE_MAP, tex);
glTexStorage2D(GL_TEXTURE_CUBE_MAP, 10, GL_RGBA8, 1024, 1024);

// Store texture data into its texel array
for (int face = 0, face < 6; facc++)
{
	GLenum target = GL_TEXTURE_CUBE_MAP_POSITIVE_X + face;
	glTexSubImage2D(target, // Face
						0, // Level
						0, 0, // X, Y offset
						1024, 1024, // Size of face
						GL_RGBA, // Format
						GL_UNSIGNED_BYTE, // Type
						texture_data[face]); // Data
}
```

**Using Cube Maps for Environment Mapping**

- Environmental mapping is a common use for cube-map texture
- Calculate the texture coordinate from which to sample the cube map by reflecting the incoming view vector around the surface normal at the point to be textured
- The vertex shader transforms the object space position into view space by multiplying it by a concatenated model-view-projection matrix
- Rotates the surface normal into view space by multiplying it by a concatenated model-view matrix

_Cube-Map Environment Mapping Example - Vertex Shader_
```c
#version 330 vore
layout (location = 0) in vec3 in_positon;
layout (location = 1) in vec3 in_normal;

// Surface normal and view-space position
out vec3 vs_fs_normal;
out vec3 vs_fs_position;

// Model-view-projection and model-view matrices
uniform mat4 mat_mvp;
uniform mat4 mat_mv;

void main(void)
{
	// Clip-space position
	gl_Position = mat_mvp * in_position;
	// View-space normal and position
	vs_fs_normal = mat3(mat_mv) * in_normal;
	vs_fs_position = (mat_mv * in_position).xyz;
}
```

- `reflect` bounces the view vector off the surface and into the cube map

_Cube-Map Environment Mapping Example - Fragment Shader_
```c
#version 330 core

// Surface normal and view-space position
in vec3 vs_fs_normal;
in vec3 vs_fs_position;

// Final fragment color
layout (location = 0) out vec4 color;

// Cube-map texture
uniform samplerCube tex;

void main(void)
{
	vec3 tc = reflect(-vs_fs_position, normalie(vs_fs_normal));
	color = vec4(0.3, 0.2, 0.1, 1.0) +
			vec4(0.97, 0.83, 0.79, 0.0) *
			texture(tex, tc);
}
```

# Linear Filtering

- Linear filtering is a technique in which a coordinate is used to select adjacent samples from a discretely sampled signal and replace that signal with a linear approximation of the original

![[Pasted image 20250923081754.png]]


- As long as the sampling rate (resolution) of the texture is high enough relative to the sharp peaks in the image data (details), a linear reconstruction of the image will appear to have high quality
- Coordinates will find the two samples that lie closest to it
- Create weights for each of the samples and then create a weighted average
- Linear resampling is separable
	- A separable operation is one that can be deconstructed into two or more, usually similar passes over the data

- Filtering options are controlled by the texture filter modes
- The sampler object represents a collection of parameters that control how texels are read from textures
