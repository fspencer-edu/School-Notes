# Divide-and-Conquer: Constructing Decision Trees


- The problem of constructing a decision tree can be expressed recursively
- The measure of purity that is used is called the information and is measured in units called bits
- The process terminates when all leaf nodes are pure; when they contain instances that all have the same classification

## Calculating Information

- When the number of either yes's or no's is zero, the information is zero
- If equal, the information reaches a max
- There is only one function that satisfies all these properties, known as the information value or entropy

## Highly Branching Attributes

- Branching on ID code produces the tree stump
- A modification of the measure called the gain ratio is used
- The gain ratio is derived by taking into account the number and size of daughter nodes into which an attribute splits the dataset
- A divide-and-conquer approach to decision tree induction, sometimes called top-down induction of decision trees, was developed and refined

# Covering Algorithms: Constructing Rules

- An alternative approach is to take each class in turn and seek a way of covering all instances of it, an excluding instances not in the class
	- Covering
- This leads to a set of rules rather than a decision tree

## Rules vs. Trees

- Whereas the covering algorithm is concerned only with covering a single class, the decision would take both classes into account
- In the multi-class case, a decision tree split takes all classes into account in trying to max the purity of the split, whereas the rule-generating method concentrates on one class at a time

## A Simple Converting Algorithm

- Covering algorithms operate by adding test to the rule that is under construction
- Divide-and-conquer algorithms operate by adding tests to the tree that is under construction

## Rules vs. Decision Lists

- Order-independent rules provide more modularity by acting independently
- Methods, such as PRISM, can be described as separate-and-conquer algorithms
