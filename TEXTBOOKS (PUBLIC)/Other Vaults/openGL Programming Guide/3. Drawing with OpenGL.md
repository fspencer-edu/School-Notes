- The primary use of OpenGL is to render graphics into a framebuffer
- Complex objects are broken into primitives
	- Points
	- Lines
	- Triangles

# OpenGL Graphics Primitives

- Line and triangle types can be combined together to form strops, loops, and fans
- Points, lines, and triangles are the native primitive types
- Other types include patches

# Points

- Points are represented by a single vertex
- The vertex represent a point in 4D homogeneous coordinates
- OpenGl determines which pixels are covered by the point using a set of rules called rasterization rules
- A sample is considered covered by a point if it falls within a square centred on the point's location in window coordinates

`void glPointSize(GLfloat size)`
- Sets the fixed size, in pixels, that will be used for points when GL_PROGRAM_POINT_SIZE is not enabled

- The default point size is 1.0
- If the pixel is increased, then each point vertex may end up lighting more than one pixel

**Point Sprites**

- The fragment shader is run for every fragment in the point
- Fragment shaders include a special built-in variable, `gl_PointCoord`, which contains the coordinate within the point where the current fragment is located

# Lines, Strips, and Loops

- The term line refers to a line segment
- Lines are represented by pairs of vertices, one for each endpoint
- Lines can be joined together to represent a connected series of line segments
- A closed sequence is known as a line loop, where a open sequences is a line strip
- Lines have no area
- The rule for line rasterization is known as the diamond exit rule
	- A line from A to B, a pixel should be lit if the line passes through the imaginary edge of a diamond shape drawn inside the pixel's square area on the screen

`void glLineWidth(GLfloat)`
- Sets the fixed width of line
- Default is 1.0
- Must be greater than 0.0

- When anti-aliasing is turned on, lines are treated as rectangles aligned along the line

# Triangles, Strips, and Fans

- Triangles are made up a collection of 3 vertices
- A triangle is rendered by projecting each of the three vertices into screen space and forming three edges running between the edges

Pixels that share an edge
- No pixel on a shared edge between two triangles that together would cover the pixel should be left unlit
- No pixel on a shared edge between two triangle should be lit by more than one of them

- Rasterize meshes with shared edges without gaps between the triangles, and without overdraw
- When a triangle strip is rendered, the first 3 vertices form the first triangle, then each subsequent vertex forms another triangle along the last two vertices of the previous triangle

- When rendering a triangle fan, the first vertex forms a shared point that is included in each subsequent triangle
- Triangles are then formed using that shared point and the next two vertices

## Restarting Primitives

- OpenGL has the ability to restart primitives within the same drawing command with primitive restart index
- New rendering primitive of the same type is started with the vertex following the index

`void glPrimitiveRestartIndex(GLuint index)`
- Specifies the vertex array element index used to indicate that a new primitive should be started during rendering

- The primitive restart index has set to 8
- As the triangles are rendered, OpenGL watches for the index 8 to be read from the element array buffer, and when it sees it go by, rather than creating a vertex, it ends the current triangle strip

## Using the Instance Counter in Shaders

- In addition to the instanced vertex attributes, the index of the current instance is available to the vertex shader in the built-in variable, `gl_InstanceID`
- Variable is implicitly declared as an integer
- Starts counting from 0, and goes up by 1 every time an instance is rendered
	- Index uniform arrays
	- Perform texture lookups
- Vertex attributes are replaced with texture buffer objects (TBOs) lookups
- Instead, a first TBO containing colour, and a second TBO containing the model matrices are created for each instance

_`gl_VertexID` Example Vertex Shader
```c
// View matrix and the projection matrix
uniform mat4 view_matrix;
uniform mat4 projection_matrix;

// TBOs
uniform sampleBuffer color_tbo;
uniform sampleBuffer model_matrix_tbo;

// Output of the vertex shader matech to the fragment shader
out VERTEX
{
	vec3 normal;
	vec4 color;
} vertex;

void main(void){
	// Use gl_InstanceID to obtain the instance color from the TBO
	vec4 color = texelFetch(color_tbo, gl_InstanceID);

	// Matrix model
	vec4 col1 = texelFetch(model_matrix_tbo, gl_InstanceID * 4);
	vec4 col2 = texelFetch(model_matrix_tbo, gl_InstanceID * 4 + 1);
	vec4 col3 = texelFetch(model_matrix_tbo, gl_InstanceID * 4 + 2);
	vec4 col4 = texelFetch(model_matrix_tbo, gl_InstanceID * 4 + 3);

	// Assemble 4 columns into a matrix
	mat4 model_matrix = mat4(col1, col2, col3, col4);

	// Construct a model_view matrix from the uniform
	mat4 model_view_matrix = view_matrix * model_matrix;

	// Transform position
	gl_Postion = projection_matrix * (model_view_matrix * position);

	// Transform the normal
	vertex.normal = mat3(model_view_matrix) * normal;

	// Pass the per-instance color through the fragment shader
	vertex.color = color;
}
```
